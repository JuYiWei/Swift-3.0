属性

Attributes 官方文档

属性提供了有关声明或类型的更多信息。有两种属性中夫特，那些适用于声明和那些适用于类型。

您可以通过写指定属性@的符号后跟属性的名称和属性接受任何参数：

@ 属性名
@ 属性名（属性参数）
一些声明属性接受指定有关属性以及如何适用于特定的声明更多信息参数。这些属性参数包含在括号中，其格式是由他们所属的属性定义。

声明属性

你可以申请一个声明属性，只声明。

available
应用此属性的任何声明，表示该声明的生命周期相对于某些平台和操作系统版本。

该available属性总是与两个或多个以逗号分隔的属性参数列表出现。这些争论开始与下列平台名称之一：

iOS
iOSApplicationExtension
macOS
macOSApplicationExtension
watchOS
watchOSApplicationExtension
tvOS
tvOSApplicationExtension
您也可以使用星号（*）来表示对上面列出的所有平台名称申报的可用性。

其余的参数可以按任何顺序出现，并指定有关声明的生命周期的其他信息，包括重要的里程碑。

该unavailable参数表示该声明不提供指定的平台上。
该introduced参数表示在其中引入了声明中的指定平台的第一个版本。它有以下形式：
介绍：版本号
的版本号由一个或多个正整数，由句点分隔的。
该deprecated参数表示指定的平台，在该声明已被否决的第一个版本。它有以下形式：
弃用：版本号
可选的版本号由一个或多个正整数，由句点分隔的。省略版本号表示该声明目前正在过时，无需提供有关发生的废弃时的任何信息。如果省略了版本号，省略冒号（:）也是如此。
该obsoleted参数表示指定的平台，在该声明被废弃的第一个版本。当一个声明过时，它是从指定的平台中清除，不能再使用。它有以下形式：
过时：版本号
的版本号由一个或多个正整数，由句点分隔的。
该message参数用于提供的编译器发出关于使用取消或已过时声明警告或错误时显示的文本消息。它有以下形式：
消息：消息
该消息由一个字符串的。
该renamed参数用于提供指示的新名称为一个已经更名声明的文本消息。新的名称是由编译器发出关于使用更名声明的错误时显示。它有以下形式：
更名为：新名称
该新名称由字符串文本。
您可以使用renamed参数与配合unavailable参数和一个类型别名声明，表示该声明已被重命名你的代码的客户。例如，当一个声明的名称是一个框架或库的版本之间的变化，这是很有用的。
// First release
protocol MyProtocol {
// protocol definition
}
// Subsequent release renames MyProtocol
protocol MyRenamedProtocol {
// protocol definition
}

@available(*, unavailable, renamed: MyRenamedProtocol)
typealias MyProtocol = MyRenamedProtocol
您可以将多个available单一声明属性在不同的平台指定声明的可用性。编译器使用的available，只有当属性指定当前的目标平台相匹配的平台属性。

如果某个available属性只指定了一个introduced除平台名称参数参数，以下简写语法来代替：

@available（平台名称 版本号，*）
为简化语法available属性使得可用性进行精确表达的多种平台。尽管两种形式是相同的功能，简写形式是首选只要有可能。

@available(iOS 10.0, macOS 10.12, *)
class MyClass {
// class definition
}
discardableResult
应用此属性的函数或方法声明来抑制编译器警告时返回一个值的函数或方法被调用，而无需使用其结果。

GKInspectable
应用此属性，露出一个自定义GameplayKit组件属性的SpriteKit编辑器UI。

objc
应用此属性，可以在Objective-C，例如表示任何声明，非嵌套类，协议，非泛型枚举（受限于整数原始值类型），属性和方法（包括getter和setter）类和协议，初始化，deinitializers，和下标。该objc属性告诉编译器，一个声明提供Objective-C代码使用。

标有类objc属性必须在Objective-C中定义的类继承。如果你申请的objc属性类或协议，它含蓄地应用于类或协议的Objective-C的兼容成员。编译器还含蓄地添加objc属性，从 ​​标有另一个类继承的类objc属性或在Objective-C中定义的类。标有协议的objc属性不能从没有协议继承。

如果你申请的objc属性为枚举，每个枚举情况下，暴露在Objective-C代码为枚举名称的串联和案件名称。案件名字的第一个字母大写。例如，一个名为案件venus在斯威夫特Planet枚举暴露在Objective-C代码为命名的情况PlanetVenus。

该objc属性选择性地接受一个属性的参数，它由一个标识符。当你要暴露一个不同的名称的Objective-C的实体使用该属性objc属性应用于。您可以使用此参数名称类，枚举，枚举的情况下，协议，方法的getter，setter以及初始化。下面的例子揭示了吸气enabled的财产ExampleClass，以Objective-C代码isEnabled，而不是仅仅作为属性本身的名称。

@objc
class ExampleClass: NSObject {
var enabled: Bool {
@objc(isEnabled) get {
// Return the appropriate value
}
}
}
nonobjc
应用此属性的方法，属性，下标，或初始化声明以抑制隐性objc属性。该nonobjc属性告诉编译器作出声明的Objective-C代码可用，即使它可能代表它在Objective-C。

您可以使用nonobjc属性来解析圆弥合在标有一类方法objc的属性，并允许的方法和初始化超载在标有一类objc属性。

标有一种方法，nonobjc属性不能覆盖打上一个方法objc属性。然而，标有一个方法objc属性可以覆盖打上一个方法nonobjc属性。类似地，方法标有nonobjc属性不能满足标明的方法的协议的要求objc属性。

NSApplicationMain
应用此属性的一类，表明它是应用程序委托。使用此属性等同于调用的NSApplicationMain(::)函数。

如果不使用该属性，提供一个main.swift与调用的顶级代码文件NSApplicationMain(::)如下功能：

import AppKit
NSApplicationMain(CommandLine.argc, CommandLine.unsafeArgv)
NSCopying
应用此属性类的存储变量属性。此属性将导致财产的setter与一个合成副本属性的值，返回用的copyWithZone(_:)方法，而不是物业本身的价值。该属性的类型必须符合NSCopying协议。

该NSCopying属性作用类似Objective-C的方式copy财产属性。

NSManaged
应用该属性的实例方法或从继承的类的存储变量属性NSManagedObject表明，核心数据动态提供其在运行时实现，基于相关实体描述。对于标有一个属性NSManaged的属性，核心数据也提供了在运行时的存储。应用这一属性也意味着objc属性。

testable
应用此属性import的声明与测试编译的模块使访问标有任何实体的internal访问级别修饰符，如果他们与申报的public访问级别修饰符。测试还可以访问类和标有集体成员internal或public访问级别修饰符如同它们与声明open访问级别修饰符。

UIApplicationMain
应用此属性的一类，表明它是应用程序委托。使用这个属性相当于调用UIApplicationMain函数并传递这个类的名字作为委托类的名称。

如果不使用该属性，提供一个main.swift在调用顶层的代码文件UIApplicationMain(:::)的功能。例如，如果您的应用程序使用的自定义子类UIApplication为主要阶级，调用UIApplicationMain(:::)使用这个属性的函数。

此属性用于自动包装在一个无参数的闭包表达式推迟表达式的评价。应用此属性的参数的类型方法或函数的声明，不带任何参数，并返回表达式的类型的值的函数类型的参数。对于如何使用的示例autoclosure属性，参见Autoclosures和功能类型。
