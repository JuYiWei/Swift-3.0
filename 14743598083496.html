简介

Class and Structures 官方文档

  • Swift的类和结构，几乎是一样的，都是构成程序的重要组成。

比较类和结构

类和结构共同点：

定义属性来存储值
限定的方法来提供功能
定义标用标语法，提供访问其值
定义初始化设置它们的初始状态
延伸拓展其功能超出了默认的实现
顺应协议来提供某一种标准功能
欲了解更多信息，请参见属性，方法，下标，初始化，扩展和协议。

类比结构附加的功能：

继承允许一个类继承的另一个特点。
类型转换可以检查并在运行时解释一个类实例的类型。
Deinitializers使类的一个实例，以释放它已经分配的任何资源。
引用计数允许多个参考一类实例。

欲了解更多信息，请参见继承，类型转换，反初始化，并自动引用计数。

定义语法

struct Resolution {
    var width = 0
    var height = 0
}

class VideoMode {
    var resolution = Resolution()
    var interlaced = false
    var frameRate = 0.0
    var name: String?
}

类与结构实例

  • 实例化方法。
let someResolution = Resolution()
let someVideoMode = VideoMode()

访问属性

  • 点语法访问属性，支持链式。
let w = someResolution.width
let w2 = someVideoMode.resolution.width

结构初始化2

  • 使用类似CGRect，是类没有的。
let vga = Resolution(width: 640, height: 480)

结构和枚举是值类型

  • 值类型，即传值时copy新的一份。
let hd = Resolution(width: 1920, height: 1080)
var cinema = hd
cinema.width = 2048
  • 枚举，以及所有基础数据类型（String，Int，Bool...）都是值类型，举例略。

类是引用类型

  • 不拷贝，直接使用原有类。
let tenEighty = VideoMode()
tenEighty.resolution = hd
tenEighty.interlaced = true
tenEighty.name = "1080i"
tenEighty.frameRate = 25.0
  • 检查与值类型的区别。
let alsoTenEighty = tenEighty
alsoTenEighty.frameRate = 30.0

检查类实例

  • 类的实例比较，不同于基础数据类型的==和!=，而是使用===和!==。
class Resolution {
    var width = 0
    var height = 0
}

let a = Resolution()
let b = Resolution()
let c = b

if a === b {
    print("a === b")
}

if b === c {
    print("b === c")
}

指针

  • Swift 的常量和变量都类似于指针，但并不是直接指向内存地址，也不需要写*。总体看来，Swift一切皆对象。

类和结构的选择使用

  • 结构的主要目的是封装几个比较简单的数据值。对比OC的CGRect，NSRange，CGSize等。
  • 大多数自定义数据结构应该是类，而不是结构。

字符串，数组和字典都是结构

  • 值类型，copy。不多说了。
