泛型参数和参数

本章介绍了泛型类型，函数和初始化参数和参数。当你声明一个泛型类型，函数或初始化，您可以指定类型参数的泛型类型，函数或初始化程序可以工作。这些类型参数作为由实际的具体类型参数替换创建了一个泛型类型的实例或一个泛型函数或初始化程序被调用时占位符。

对于仿制药的斯威夫特概述，请参见泛型。

泛型参数条款

一个泛型参数子句指定泛型类型或函数的类型参数，对这些参数的任何相关的约束和要求一起。泛型参数子句中的尖括号（<>），并具有以下形式：

< 通用参数表 >

类型参数：约束
泛型参数由一个的类型参数后跟一个可选的约束。一个类型参数仅仅是一个占位符类型的名称（例如T，U，V，Key，Value，等）。你必须在类型，函数或初始化声明的其余部分访问类型参数（及任何与其相关的类型），包括在功能或初始化的签名。

该约束指定类型参数从一个特定的类继承或符合协议或协议组成。例如，在下面的通用功能，泛型参数T: Comparable表示的类型参数取代任何类型的参数T必须符合Comparable协议。

func simpleMax(_ x: T, _ y: T) -> T {
if x < y {
return y
}
return x
}
因为Int与Double，例如，既符合Comparable协议，该函数接受任一类型的参数。在泛型类型相反，当你使用一个通用的函数或初始化你不指定一个通用的说法条款。该类型的参数，而不是从传递给函数或初始化参数的类型推断。

simpleMax(17, 42) // T is inferred to be Int
simpleMax(3.14159, 2.71828) // T is inferred to be Double

该要求在通用where条款指定类型参数从类继承或符合协议或协议组成。虽然通用where条款为对类型参数的简单的约束提供了语法糖（例如，相当于 where T: Comparable等等），你可以用它来 ​​提供类型参数及其相关类型更为复杂的约束。例如，您可以约束相关类型的类型参数符合协议。例如， where S.Iterator.Element: Equatable指定S符合Sequence协议和相关类型S.Iterator.Element符合Equatable协议。这个约束确保了序列中的每个元素是equatable。

一般的参数条款

一个一般的参数子句指定泛型类型的类型参数。一个通用的说法子句在尖括号（<>），并具有以下形式：

< 通用参数列表 >
在通用的参数列表是一个逗号分隔的类型参数列表。一个类型参数是一个实际的具体类型替换泛型类型的泛型参数子句中相应类型参数的名称。其结果是，一般类型的一个专门的版本。下面的例子显示了雨燕标准库的通用字典类型的简化版本。

struct Dictionary: Collection, ExpressibleByDictionaryLiteral {
/* ... */
}
通用的专门版本Dictionary类型，Dictionary是通过更换通用参数形成Key: Hashable并Value与混凝土类型参数String和Int。每种类型的参数必须满足它取代了泛型参数的所有限制，包括在通用规定的任何附加要求where条款。在上面的例子中，Key类型参数被约束为符合Hashable协议，因此String也必须符合Hashable协议。

你也可以更换一个类型参数本身就是一个泛型类型的专用版本（只要它满足相应的约束和要求）类型参数。例如，您可以替换类型参数Element在Array同一个阵列的专用版本，Array以形成一个数组的元素本身是整数数组。
