Strings and Characters 官方文档

  • 字符串是一系列字符，如hello, world或albatross。Swift字符串类似是String，可以用各种方式进行访问，包括Character值。

  • Swift的String和Character类型，快速高效，现代化，兼容Unicode。字符串创建和操作的语法类似C语言，具有轻量级和可读性的特点。字符串连接使用+运算符即可。字符串可变性使用常量和变量区分即可。还可以使用字符串插入常量，变量，表达式，称为字符串插值法。

  • Swift的String类型与NSString类无缝桥接。可以访问NSString方法，并无需进行转换。

  • 有关使用的详细信息String与Foundation和Cocoa，参考Using Swift with Cocoa and Objective-C (Swift 3)

字符串字面量

  • 字符串是一对双引号包围的文本""。Swift默认类型推断为String类型的常数。
let someString = "Some string literal value"

初始化一个空字符串

  • 初始化一个空的字符串。
var emptyString = ""               // empty string literal
var anotherEmptyString = String()  // initializer syntax
  • 判断String是否为空。
if emptyString.isEmpty {
    print("Nothing to see here")
}

字符串可变性

  • 字符串的可变性，现在统一由常量与变量控制。与OC不一样，不再区分为NSString与NSMutableString。
let constantString = "Highlander"

var variableString = "Horse"
variableString += " and carriage"

字符串是值类型

  • Swift的String类型是值类型。简单的说就是传递值时，是拷贝一份新值再用于传递。原字符串并不改变。Swift编译器也对字符串进行了优化，实际复制只在必要时才处理，得以出色表现。

使用字符

  • String通过遍历其characters属性，可以访问Character的值。
for character in "Dog!🐶".characters {
    print(character)
}
  • 可以创建一个单独Character，并可以转换String类型。
let exclamationMark: Character = "!"

let catCharacters: [Character] = ["C", "a", "t", "!", "🐱"]
let catString = String(catCharacters)
print(catString)

连接字符串和字符

  • String使用+来按顺序拼接字符串。
let string1 = "hello"
let string2 = " there"
var welcome = string1 + string2
  • String在现有String值的基础上拼接+=。
var instruction = "look over"
instruction += string2
  • String类型提供append()方法
let exclamationMark: Character = "!"
welcome.append(exclamationMark)

字符串插值

  • 也就是曾经的格式化字符串，使用\(some)来插入字符串，可以插入各种类型各种表达式。
let multiplier = 3
let message = "\(multiplier) times 2.5 is \(Double(multiplier) * 2.5)"
  • 括号内不能包含反斜杠\，回车，换行。

Unicode

  • Swift的String和Character类型完全支持Unicode兼容。

  • 字符串中的特殊字符

转义特殊字符
\0（空字符） 
\\（反斜杠） 
\t（水平制表） 
\n（换行） 
\r（回车） 
\"（双引号）
\'（单引号）
  • 任意Unicode标量，写成\u{}，其中值为1-8位十六进制数。
let wiseWords = "\"Imagination is more important than knowledge\" - Einstein"

let dollarSign = "\u{24}"        // $,  Unicode scalar U+0024
let blackHeart = "\u{2665}"      // ♥,  Unicode scalar U+2665
let sparklingHeart = "\u{1F496}" // 💖, Unicode scalar U+1F496

  • 扩展字元簇

  • Swift的每个Character类型代表一个扩展字元簇。由一个或多个的Unicode标量结合组成单一可读字符。

  • é可以被表示为单一的Unicode（LATIN SMALL LETTER E WITH ACUTE或U+00E9）。可以被表示为一对标量-一个标准信e（LATIN SMALL LETTER E或U+0065），接着是COMBINING ACUTE ACCENT标量（U+0301）。

let eAcute: Character = "\u{E9}"                         // é
let combinedEAcute: Character = "\u{65}\u{301}"          // e followed by ́
  • 朝鲜字母韩文音节可以表示为任一预组合或分解序列。
let precomposed: Character = "\u{D55C}"                  // 한
let decomposed: Character = "\u{1112}\u{1161}\u{11AB}"   // ᄒ, ᅡ, ᆫ ==> 한
  • 封闭标记（如COMBINING ENCLOSING CIRCLE，或U+20DD）包括其它的Unicode标量作为一个单一的 一部分Character价值：
let enclosedEAcute: Character = "\u{E9}\u{20DD}"
// enclosedEAcute is é⃝
  • 区域指示器符号的Unicode标量可以成对使一个单一的组合Character值，如该组合REGIONAL INDICATOR SYMBOL LETTER U（U+1F1FA）和REGIONAL INDICATOR SYMBOL LETTER S（U+1F1F8）：
let regionalIndicatorForUS: Character = "\u{1F1FA}\u{1F1F8}"
// regionalIndicatorForUS is 🇺🇸

字符计数

  • 检索字符串中Character的数量，可以使用characterss属性的count属性。
let unusualMenagerie = "Koala 🐨, Snail 🐌, Penguin 🐧, Dromedary 🐪"
print("unusualMenagerie has \(unusualMenagerie.characters.count) characters")
  • Swift的扩展字形集群的Character值，字符串连接和修改可能并不影响一个字符串的字符数。例如，如果你初始化字符串cafe，然后追加一个COMBINING ACUTE ACCENT（U+0301）的字符串，第四个字符é，而不是e，得到的字符串字符数仍然是4，
var word = "cafe"
print("the number of characters in \(word) is \(word.characters.count)")
// Prints "the number of characters in cafe is 4"
 
word += "\u{301}"    // COMBINING ACUTE ACCENT, U+0301
 
print("the number of characters in \(word) is \(word.characters.count)")
// Prints "the number of characters in café is 4"
  • 与NSString的length的长度计算方式不同。

访问和修改字符串

  • 访问和修改的字符串，使用其方法和属性，或使用下标语法。

字符串索引

  • 每个String值具有相关联的索引类型String.Index，对应字符串中每一个的位置Character。由于Swift的字符串计数所讲的情况，所以不能直接使用整数索引。

  • 使用startIndex属性来访问第一的位置Character。endIndex是最后一个Character。endIndex属性不是一个字符串的下标有效的参数。如果String是空的，startIndex和endIndex是相等的。

  • 使用索引之前和之后的方法访问index(before:)和index(after:)截取部分字符串。可以使用index(_:offsetBy:)截取偏移后的字符串。

  • 使用标语法来访问Character在一个特定的String索引。

let greeting = "Guten Tag!"
greeting[greeting.startIndex]
// G
greeting[greeting.index(before: greeting.endIndex)]
// !
greeting[greeting.index(after: greeting.startIndex)]
// u
let index = greeting.index(greeting.startIndex, offsetBy: 7)
greeting[index]
// a
  • 索引越界，将触发运行时错误。
greeting[greeting.endIndex] // error
greeting.index(after: endIndex) // error
使用indices的财产characters属性来访问字符串中的所有单个字符的指标。

for index in greeting.characters.indices {
    print("\(greeting[index]) ", terminator: "")
}
// Prints "G u t e n   T a g ! "
  • startIndex与endIndex属性和index(before:)，index(after:)和index(_:offsetBy:)，符合任何遵守Collection协议的类型。包括String，以及集合类型如Array，Dictionary，和Set。

插入和取出

  • 要在指定索引处插入一个字符，使用insert(_:at:)，插入另一个字符串，使用insert(contentsOf:at:)。
var welcome = "hello"
welcome.insert("!", at: welcome.endIndex)
// welcome now equals "hello!"
 
welcome.insert(contentsOf:" there".characters, at: welcome.index(before: welcome.endIndex))
// welcome now equals "hello there!"
  • 要在指定索引处移除一个字符，使用remove(at:)，指定范围内删除字符串，使用removeSubrange(_:)。
welcome.remove(at: welcome.index(before: welcome.endIndex))
// welcome now equals "hello there"
 
let range = welcome.index(welcome.endIndex, offsetBy: -6)..<welcome.endIndex
welcome.removeSubrange(range)
// welcome now equals "hello"
  • 使用的insert(_:at:)，insert(contentsOf:at:)，remove(at:)，和removeSubrange(_:)，符合任何遵守RangeReplaceableCollection协议的类型。包括String，以及集合类型如Array，Dictionary，和Set。

比较字符串

  • Swift提供了三种方式来比较文本值：字符串和字符相等，前缀和后缀的相等。

字符串和字符比较

  • 字符串和字符的比较使用==和!=。
let quotation = "We're a lot alike, you and I."
let sameQuotation = "We're a lot alike, you and I."
if quotation == sameQuotation {
    print("These two strings are considered equal")
}
// Prints "These two strings are considered equal"
  • 比较时，具有相同的语意和外观，使用字元簇组成的与某个标量的语意和外观相同，则被视为相同。
例如，LATIN SMALL LETTER E WITH ACUTE（U+00E9）是规范等价于LATIN SMALL LETTER E（U+0065），随后加入COMBINING ACUTE ACCENT（U+0301）。这两个扩展字形簇是有效的方法来表示的字符é，因此它们被认为是规范等价：

// "Voulez-vous un café?" using LATIN SMALL LETTER E WITH ACUTE
let eAcuteQuestion = "Voulez-vous un caf\u{E9}?"
 
// "Voulez-vous un café?" using LATIN SMALL LETTER E and COMBINING ACUTE ACCENT
let combinedEAcuteQuestion = "Voulez-vous un caf\u{65}\u{301}?"
 
if eAcuteQuestion == combinedEAcuteQuestion {
    print("These two strings are considered equal")
}
// Prints "These two strings are considered equal"
  • 不同的标量则都视为不同。
LATIN CAPITAL LETTER A（U+0041或"A"），如在英国使用的，是不等同于CYRILLIC CAPITAL LETTER A（U+0410或"А"），如在俄罗斯使用。字符看上去很相似，但不具有相同的语言意义：

let latinCapitalLetterA: Character = "\u{41}"
 
let cyrillicCapitalLetterA: Character = "\u{0410}"
 
if latinCapitalLetterA != cyrillicCapitalLetterA {
    print("These two characters are not equivalent.")
}
// Prints "These two characters are not equivalent."
  • Swift字符串比较不区分语言环境。

前缀和后缀比较

  • 检查一个字符串是否具有特定字符串前缀或后缀，使用字符串的hasPrefix(_:)和hasSuffix(_:)。
let romeoAndJuliet = [
    "Act 1 Scene 1: Verona, A public place",
    "Act 1 Scene 2: Capulet's mansion",
    "Act 1 Scene 3: A room in Capulet's mansion",
    "Act 1 Scene 4: A street outside Capulet's mansion",
    "Act 1 Scene 5: The Great Hall in Capulet's mansion",
    "Act 2 Scene 1: Outside Capulet's mansion",
    "Act 2 Scene 2: Capulet's orchard",
    "Act 2 Scene 3: Outside Friar Lawrence's cell",
    "Act 2 Scene 4: A street in Verona",
    "Act 2 Scene 5: Capulet's mansion",
    "Act 2 Scene 6: Friar Lawrence's cell"
]
  • 可以使用hasPrefix(_:)方法与romeoAndJuliet数组，计算剧中场景1的数量。
var act1SceneCount = 0
for scene in romeoAndJuliet {
    if scene.hasPrefix("Act 1 ") {
        act1SceneCount += 1
    }
}
print("There are \(act1SceneCount) scenes in Act 1")
// Prints "There are 5 scenes in Act 1"
  • 使用hasSuffix(_:)方法，计算表演地点Capulet's mansion的数量。
var mansionCount = 0
var cellCount = 0
for scene in romeoAndJuliet {
    if scene.hasSuffix("Capulet's mansion") {
        mansionCount += 1
    } else if scene.hasSuffix("Friar Lawrence's cell") {
        cellCount += 1
    }
}
print("\(mansionCount) mansion scenes; \(cellCount) cell scenes")
// Prints "6 mansion scenes; 2 cell scenes"
  • hasPrefix(_:)和hasSuffix(_:)，逐个字符进行比较。

字符串编码

准备字符串，下面开始操作。

let dogString = "Dog‼🐶"

UTF-8表示

  • 通过遍历其utf8属性访问String的UTF-8值。此属性是类型String.UTF8View，也就是UInt8值。
for codeUnit in dogString.utf8 {
    print("\(codeUnit) ", terminator: "")
}
print("")
// 68 111 103 226 128 188 240 159 144 182
  • [0，3)位十进制codeUnit值（68，111，103）代表的字符D，o和g，其UTF-8表示的是相同的ASCII表示。
  • [3，6)位十进制codeUnit值（226，128，188）是一个三字节的UTF-8表示DOUBLE EXCLAMATION MARK的字符。
  • [6,10)位十进制codeUnit值（240，159，144，182）是一个四字节UTF-8表示DOG FACE的字符。

UTF-16表示

  • 通过遍历其utf16属性访问String的UTF-16值。此属性是类型String.UTF16View，也就是UInt16值。
for codeUnit in dogString.utf16 {
    print("\(codeUnit) ", terminator: "")
}
print("")
// Prints "68 111 103 8252 55357 56374 "
  • [0，3)位十进制codeUnit值（68，111，103）所 ​​表示的字符D，o以及g，其UTF-16代码单元的作为字符串的UTF-8表示相同的值（因为这些Unicode标量代表ASCII字符）。
  • [3，4)位十进制codeUnit值（8252）是十六进制值的十进位等值203C，它代表了Unicode标U+203C为DOUBLE EXCLAMATION MARK字符。该字符可被表示为UTF-16中的单一代码单元。
  • [4，6)位十进制codeUnit值（55357和56374）是一个UTF-16代理对代表性DOG FACE人物。这些数值的高代理值U+D83D（十进制值55357）和低代理值U+DC36（十进制值56374）。

Unicode标表示

  • 通过遍历其unicodeScalars属性访问String的Unicode值。此属性是类型UnicodeScalarView，也就是UnicodeScalar值。每UnicodeScalar有一个value返回21位标量，也就是UInt32值。
for scalar in dogString.unicodeScalars {
    print("\(scalar.value) ", terminator: "")
}
print("")
// Prints "68 111 103 8252 128054 "
  • [0，3)位UnicodeScalar（68，111，103）再次表示字符D，o和g。
  • [3，4)位十进制codeUnit值（8252）再次是十六进制值的十进位等值203C，它代表了Unicode标U+203C为DOUBLE EXCLAMATION MARK字符。

  • [4，5)位十进制codeUnit值（128054）是十六进制值的十进制等效1F436，它代表了Unicode标U+1F436为DOG FACE字符。

  • 作为替代查询他们的value属性，每一个UnicodeScalar值也可以被用来构造一个新String值。

for scalar in dogString.unicodeScalars {
    print("\(scalar) ")
}
// D
// o
// g
// ‼
// 🐶
