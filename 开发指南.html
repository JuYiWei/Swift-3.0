常量和变量

常量和变量声明

  • 在使用之前，常量和变量必须先声明。常量let和变量var。
let maximumNumberOfLoginAttempts = 10
var currentLoginAttempt = 0
  • 可以在一行声明多个常量（变量），用逗号分隔。
var x = 0.0, y = 0.0, z = 0.0

类型标识

  • 当你声明一个常量（变量），可以指定类型。但不是必须的，使用初始化值初始化时，可以自动类型判断。
var welcomeMessage: String
welcomeMessage = "Hello"
  • 可以在一行声明多个相同类型的常量（变量），用逗号分隔。
var red, green, blue: Double

常量和变量命名

  • 常量和变量名几乎可以包含任何字符，包括Unicode字符，但还是推荐正常命名规范。
let π = 3.14159
let 你好 = "你好世界"
let 🐶🐮 = "dogcow"

常量和变量打印

  • print打印常量（变量）的当前值，print函数有多种参数形式。
print(friendlyWelcome)
print(1,2,3,4 ,separator: "??", terminator: "000")
  • 打印格式字符，\()包含常量（变量）
print("The current value of friendlyWelcome is \(friendlyWelcome)")

注释

  • 类似C单行注释，以两个正斜杠//开始：
// This is a comment.
  • 多行注释以/*开始，并*/结束
/* This is also a comment
 but is written over multiple lines. */
  • 多行注释可以嵌套在其他多行注释。
/* This is the start of the first multiline comment.
 /* This is the second, nested multiline comment. */
 This is the end of the first multiline comment. */

分号

  • 语句不要求分号结尾。但是，如果一行上多个独立的语句需要分号分割。
let cat = "🐱"; print(cat)

整数

整数边界

  • 可以获取每个整数类型的最小值和最大值min与max属性：
let minValue = UInt8.min  // minValue is equal to 0, and is of type UInt8
let maxValue = UInt8.max  // maxValue is equal to 255, and is of type UInt8

INT

  • 在大多数情况下，不需要选择整数的特定大小。Swift提供默认整数类型Int，数据大小与当前系统有关。
在32位的平台上，Int其大小与相同Int32。
在64位的平台上，Int其大小与相同Int64。

UINT

  • Swift提供额外的整数类型UInt，数据大小与当前系统有关。
在32位的平台上，UInt其大小与相同UInt32。
在64位的平台上，UInt其大小与相同UInt64。

浮点数

  • Double 代表一个64位浮点数。
  • Float 表示32位浮点数。
  • Double具有15位的精度，Float具有6位。默认推断为Double，或者按需要选择使用。

类型安全和类型推断

  • 类型推断：定义常量（变量），初始化赋值时，可以自动推断类型。
let meaningOfLife = 42 // Int
let pi = 3.14159 // Double
let anotherPi = 3 + 0.14159 // Double
  • 类型安全：检查代码赋值时前后类型是否完全匹配。
let anotherPi2 = meaningOfLife + pi // error 因为meaningOfLife是Int，而pi是Double，不能直接计算

数值表示

  • 整数可以写为：
一个十进制数，无前缀
一个二进制数字，用0b前缀
一个八进制数，有0o前缀
一个十六进制数字，用0x前缀
  • 以下整数都表示 17
let decimalInteger = 17
let binaryInteger = 0b10001       
let octalInteger = 0o21           
let hexadecimalInteger = 0x11
  • 浮点数值可以是十进制（不带前缀），或十六进制（带有0x前缀）表示。十进制可以有一个可选的指数e，十六进制必须有一个指数p。
1.25e2 // 1.25×10^2 = 125.0
1.25e-2 // 1.25×10^-2 = 0.0125

0xFp2 // 15×2^2 = 60.0
0xFp-2 // 15×2^-2 = 3.75

let decimalDouble = 12.1875
let exponentDouble = 1.21875e1
let hexadecimalDouble = 0xC.3p0
  • 可以用额外的零填充并且可以包含下划线，以提高可读性。
let paddedDouble = 000123.456
let oneMillion = 1_000_000
let justOverOneMillion = 1_000_000.000_000_1

数值类型转换

  • 默认系统推断类型，整型 Int，浮点型 Double。

整型转换

  • 防止溢出。
let cannotBeNegative: UInt8 = -1
let tooBig: Int8 = Int8.max + 1
  • 整型不同大小类型转换。
let twoThousand: UInt16 = 2_000
let one: UInt8 = 1
let twoThousandAndOne = twoThousand + UInt16(one)

整型和浮点转换

  • 整型转浮点。
let three = 3
let pointOneFourOneFiveNine = 0.14159
let pi = Double(three) + pointOneFourOneFiveNine
  • 浮点到整型。
let integerPi = Int(pi)

类型别名

  • 类型别名为现有类型定义的替代名称。使用typealias定义。
typealias AudioSample = UInt16
var maxAmplitudeFound = AudioSample.min

布尔值

  • Swift有一个基本的布尔类型Bool。布尔值只有true和false。
let orangesAreOrange = true
let turnipsAreDelicious = false
  • 布尔值基本用在if的语句中。
if turnipsAreDelicious {
    print("Mmm, tasty turnips!")
} else {
    print("Eww, turnips are horrible.")
}
  • Swift的类型安全防止隐性转换Bool，与OC不同，下面的示例编译时会报错。
let i = 1
if i {
    // this example will not compile, and will report an error
}
  • 必须使用返回Bool类型的表达式。
let i = 1
if i == 1 {
    // this example will compile successfully
}

元组

  • 元组是多个值组成的复合值。元组中的值可以是任何类型，并且允许不同类型的组合。
let http404Error = (404, "Not Found")
  • 分解一个元组内容到单独的常量（变量），类似字典。
let (statusCode, statusMessage) = http404Error
print("The status code is \(statusCode)")
print("The status message is \(statusMessage)")
  • 如果只需要元组的部分值，用下划线_忽略不必要选项。
let (justTheStatusCode, _) = http404Error
print("The status code is \(justTheStatusCode)")
  • 从零开始索引，访问元组的各个元素值，类似数组。
print("The status code is \(http404Error.0)")
print("The status message is \(http404Error.1)")
  • 定义元组时，可以给元组的各个元素命名。
let http200Status = (statusCode: 200, description: "OK")
  • 如果元组元素有命名，可以使用元素名称来访问这些元素的值。
print("The status code is \(http200Status.statusCode)")
print("The status message is \(http200Status.description)")

可选类型

  • 可选类型代表两种可能：要么是一个值可以解析，要么没有值为nil。

nil

  • 可选变量可以设置成nil,表示不存在任何值。非可选变量不能使用nil。
var serverResponseCode: Int? = 404
serverResponseCode = nil
  • 如果可选的变量没有提供初始化值，该变量将自动设置为nil。
var surveyAnswer: String?

if语句，强制解析

  • 使用if语句直接与nil比较，确定有值时，使用!解析值。
var convertedNumber :Int?
convertedNumber = 18

if convertedNumber != nil {
    print("ok \(convertedNumber!)")
} else {
    print("no")
}

可选绑定

  • 使用可选类型值初始化其他变量或常量时，该表达式语句可以看做是Bool类型(但不真的是Bool类型)，因此可以使用if语句判断。也就是可选类型为nil时，初始化就失败，反之成功，并且该值可以在内部使用。
if let actualNumber = Int(possibleNumber) {
    print("\"\(possibleNumber)\" has an integer value of \(actualNumber)")
} else {
    print("\"\(possibleNumber)\" could not be converted to an integer")
}
  • 可以在一个if中，混合使用多个可寻绑定，以及Bool条件，但是需要逗号分隔，作用与&&一致。
if let firstNumber = Int("4"), let secondNumber = Int("42"), firstNumber < secondNumber && secondNumber < 100 {
    print("\(firstNumber) < \(secondNumber) < 100")
}
 
// 与上面的效果一致
if let firstNumber = Int("4") {
    if let secondNumber = Int("42") {
        if firstNumber < secondNumber && secondNumber < 100 {
            print("\(firstNumber) < \(secondNumber) < 100")
        }
    }
}

可选类型隐式解析

  • 类型强制解析，只不过不判断是否有值，直接主管判断有值，当nil时即报错。
let a = convertedNumber!
let b: Int! = convertedNumber
  • 隐式解析可选类型本质还是可选类型，任然可以使用!=nil去强制解析，以及可选绑定。
if assumedString != nil {
    print(assumedString)
}

if let definiteString = assumedString {
    print(definiteString)
}

错误处理

  • 当函数遇执行时到错误，抛出一个错误。然后，调用该函数者可以抓取的错误，并作出适当的处理。
func canThrowAnError() throws {
    // this function may or may not throw an error
}
  • 函数通过throws表明它可以抛出错误，函数前面加try关键字，表示该函数可以抛出错误，并使用catch捕获错误。
do {
    try canThrowAnError()
    // no error was thrown
} catch {
    // an error was thrown
}
  • do语句创建范围，允许误差传播到一个或多个catch。
func makeASandwich() throws {
    // ...
}
 
do {
    try makeASandwich()
    eatASandwich()
} catch SandwichError.outOfCleanDishes {
    washDishes()
} catch SandwichError.missingIngredients(let ingredients) {
    buyGroceries(ingredients)
}

断言

调试使用断言

  • 断言是一种运行时检查，条件不符合，那么就中断执行。发布配置是默认断言无效。
let age = -3
assert(age >= 0, "A person's age cannot be less than zero")
  • 如果需要，可以省略断言消息。
assert(age >= 0)

何时使用断言

  • 断言检查合适的方案包括：1）索引越界。2）无效值传递函数。3）等

  • 断言导致应用程序终止，提供比较不错的调试环境。断言，确保这些必要条件在开发过程中突出显示。
