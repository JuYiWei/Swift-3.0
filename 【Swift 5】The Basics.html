1 Constants and Variables

1.1 Declaring Constants and Variables

  • 定义大众化了：let：常量 var：变量
let maximumNumberOfLoginAttempts = 10
var currentLoginAttempt = 0

var x = 0.0, y = 0.0, z = 0.0

1.2 Type Annotations

  • 定义可指定类型，赋值时类型不同会报错
var welcomeMessage: String

welcomeMessage = "Hello"

var red, green, blue: Double

1.3 Naming Constants and Variables

  • 命名空间真大，中文编程吗？不建议使用
let π = 3.14159
let 你好 = "你好世界"
let 🐶🐮 = "dogcow"
  • 常量赋值报错
var friendlyWelcome = "Hello!"
friendlyWelcome = "Bonjour!"
// friendlyWelcome is now "Bonjour!"

let languageName = "Swift"
languageName = "Swift++"
// This is a compile-time error: languageName cannot be changed.

1.4 Printing Constants and Variables

  • 内容输出也比 OC 简单多了
print(friendlyWelcome)
// Prints "Bonjour!"

print("The current value of friendlyWelcome is \(friendlyWelcome)")
// Prints "The current value of friendlyWelcome is Bonjour!"

2 Comments

  • 常规注视
// This is a comment.

/* This is also a comment
but is written over multiple lines. */

/* This is the start of the first multiline comment.
 /* This is the second, nested multiline comment. */
This is the end of the first multiline comment. */

3 Semicolons

  • 一行多语句需要分号分割；但规范应该不建议一行多语句；也就是说不需要写分号了；
let cat = "🐱"; print(cat)

// Prints "🐱"

4 Integers

4.1 Integer Bounds

  • 数据边界
let minValue = UInt8.min  // minValue is equal to 0, and is of type UInt8
let maxValue = UInt8.max  // maxValue is equal to 255, and is of type UInt8

4.2 Int

自动区分 32、64 位

4.3 UInt

自动区分 32、64 位

5 Floating-Point Numbers

Float：32 位
Double：64 位（默认）

6 Type Safety and Type Inference

  • 类型安全：初始化后，类型不可变
let meaningOfLife = 42
// meaningOfLife is inferred to be of type Int

let pi = 3.14159
// pi is inferred to be of type Double
  • 类型推断：初始化类型不指定时，会自动推断类型
let anotherPi = 3 + 0.14159
// anotherPi is also inferred to be of type Double

7 Numeric Literals

  • 进制
let decimalInteger = 17
let binaryInteger = 0b10001       // 17 in binary notation
let octalInteger = 0o21           // 17 in octal notation
let hexadecimalInteger = 0x11     // 17 in hexadecimal notation
  • 科学记数法
let decimalDouble = 12.1875
let exponentDouble = 1.21875e1 // 10^1
let hexadecimalDouble = 0xC.3p0 // 2^0
  • 数据可视化，方便数位数
let paddedDouble = 000123.456
let oneMillion = 1_000_000
let justOverOneMillion = 1_000_000.000_000_1

8 Numeric Type Conversion

8.1 Integer Conversion

  • 数据越界报错
let cannotBeNegative: UInt8 = -1
// UInt8 cannot store negative numbers, and so this will report an error
let tooBig: Int8 = Int8.max + 1
// Int8 cannot store a number larger than its maximum value,
// and so this will also report an error
  • 类型强转：不同类型不能计算
let twoThousand: UInt16 = 2_000
let one: UInt8 = 1
let twoThousandAndOne = twoThousand + UInt16(one)

8.2 Integer and Floating-Point Conversion

  • Int\Float 类型转换
let three = 3
let pointOneFourOneFiveNine = 0.14159
let pi = Double(three) + pointOneFourOneFiveNine
// pi equals 3.14159, and is inferred to be of type Double

let integerPi = Int(pi)
// integerPi equals 3, and is inferred to be of type Int

9 Type Aliases

  • 类型别名
typealias AudioSample = UInt16

var maxAmplitudeFound = AudioSample.min
// maxAmplitudeFound is now 0

10 Booleans

  • true/false，且不再是 OC YES（1）/NO（0）
let orangesAreOrange = true
let turnipsAreDelicious = false

if turnipsAreDelicious {
    print("Mmm, tasty turnips!")
} else {
    print("Eww, turnips are horrible.")
}

// Prints "Eww, turnips are horrible."
  • Int 类型不能用来判断了，不再是 OC 的强转 BOOL 类型
let i = 1
if i {
    // this example will not compile, and will report an error
}

let i = 1
if i == 1 {
    // this example will compile successfully
}

11 Tuples

  • 元组，支持不同类型的集合
let http404Error = (404, "Not Found")
// http404Error is of type (Int, String), and equals (404, "Not Found")
  • 取值，使用 key-value 容器
let (statusCode, statusMessage) = http404Error
print("The status code is \(statusCode)")
// Prints "The status code is 404"
print("The status message is \(statusMessage)")
// Prints "The status message is Not Found"
  • 取值忽略
let (justTheStatusCode, _) = http404Error
print("The status code is \(justTheStatusCode)")
// Prints "The status code is 404"
  • index 取值
print("The status code is \(http404Error.0)")
// Prints "The status code is 404"
print("The status message is \(http404Error.1)")
// Prints "The status message is Not Found"
  • 定义时带 key
let http200Status = (statusCode: 200, description: "OK")

print("The status code is \(http200Status.statusCode)")
// Prints "The status code is 200"
print("The status message is \(http200Status.description)")
// Prints "The status message is OK"

12 Optionals

  • 可选类型：处理缺失情况，有值，或者没有值(nil)
let possibleNumber = "123"
let convertedNumber = Int(possibleNumber)
// convertedNumber is inferred to be of type "Int?", or "optional Int"

// 补充完整：等价于
let convertedNumber: Int? = Int(possibleNumber)
或者
let convertedNumber: Optional<Int> = Int(possibleNumber)

上述转换中：
String 可转换成 Int 时，convertedNumber = Int
String 不可转换成 Int 时，convertedNumber = nil
so
convertedNumber 此时的类型是 Optional 而不是 Int，因为它不能确定
在使用 convertedNumber 时，需要判断他的类型，也就是可选类型解析

12.1 nil

  • nil：没有值
var serverResponseCode: Int? = 404
// serverResponseCode contains an actual Int value of 404
serverResponseCode = nil
// serverResponseCode now contains no value
  • 可选类型，默认 nil
var surveyAnswer: String?
// surveyAnswer is automatically set to nil

12.2 If Statements and Forced Unwrapping

  • !=nil 的判断
if convertedNumber != nil {
    print("convertedNumber contains some integer value.")
}
// Prints "convertedNumber contains some integer value."

if convertedNumber != nil {
    print("convertedNumber has an integer value of \(convertedNumber!).")
}
// Prints "convertedNumber has an integer value of 123."

12.3 Optional Binding

  • 可选绑定，强转表达式可用作条件判断？表达式本身有 BOOL 类型？
if let actualNumber = Int(possibleNumber) {
    print("The string \"\(possibleNumber)\" has an integer value of \(actualNumber)")
} else {
    print("The string \"\(possibleNumber)\" could not be converted to an integer")
}

// Prints "The string "123" has an integer value of 123"
  • 骚操作
if let firstNumber = Int("4"), let secondNumber = Int("42"), firstNumber < secondNumber && secondNumber < 100 {
    print("\(firstNumber) < \(secondNumber) < 100")
}

// Prints "4 < 42 < 100"


if let firstNumber = Int("4") {
    if let secondNumber = Int("42") {
        if firstNumber < secondNumber && secondNumber < 100 {
            print("\(firstNumber) < \(secondNumber) < 100")
        }
    }
}

// Prints "4 < 42 < 100"

12.4 Implicitly Unwrapped Optionals

  • 强制解析，因为我知道它就是个 String，所以不判断，使用 ! 强制解析成 String
let possibleString: String? = "An optional string."
let forcedString: String = possibleString! // requires an exclamation mark


let assumedString: String! = "An implicitly unwrapped optional string."
let implicitString: String = assumedString // no need for an exclamation mark

if assumedString != nil {
    print(assumedString!)
}
// Prints "An implicitly unwrapped optional string."

if let definiteString = assumedString {
    print(definiteString)
}
// Prints "An implicitly unwrapped optional string."

强制解析可能报错，因为是自己认为一定不可能 nil，而做的强制解析，强制解析后类型正确，但依旧可能是 nil

13 Error Handling

  • throw-do-catch 形式，随大流，好开发
func canThrowAnError() throws {
    // this function may or may not throw an error
}

do {
    try canThrowAnError()
    // no error was thrown
} catch {
    // an error was thrown
}

func makeASandwich() throws {
    // ...
}


do {
    try makeASandwich()
    eatASandwich()
} catch SandwichError.outOfCleanDishes {
    washDishes()
} catch SandwichError.missingIngredients(let ingredients) {
    buyGroceries(ingredients)
}

14 Assertions and Preconditions

14.1 Debugging with Assertions

基本与 OC Assert 一致，参数条件为正确的情况

let age = -3
assert(age >= 0, "A person's age can't be less than zero.")
// This assertion fails because -3 is not >= 0.

assert(age >= 0)

if age > 10 {
    print("You can ride the roller-coaster or the ferris wheel.")
} else if age >= 0 {
    print("You can ride the ferris wheel.")
} else {
    assertionFailure("A person's age can't be less than zero.")
}

14.2 Enforcing Preconditions

  • 约等于 assert 线上环境可用，暂时不明白区别
// In the implementation of a subscript...
precondition(index > 0, "Index must be greater than zero.")
