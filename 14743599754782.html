Error Handling 官方文档

定义异常

  • Error协议，用于错误处理，并抛出错误用于抓取。
enum HttpResultError : Error {
    case NotFount
    case TimeOut
}

处理异常

  • 结合throw do try cacth，处理错误。

抛出异常 throw

  • 函数末尾标记throws表示函数支持异常捕获，内部判断异常情况，并抛出异常。
func httpHandler(a: Int) throws -> String {
   
    if a == 404 {
        throw HttpResultError.NotFount
    }
    else if a == 408 {
        throw HttpResultError.TimeOut
    }
    
    return "ok"
}

异常捕获并处理

  • do { try fun ok handler } catch error1 {error1 handler} catch error2 {error2 handler}。
do {
    try httpHandler(a: 408)
    print("ok")
} catch HttpResultError.NotFount {
    print("not found")
} catch HttpResultError.TimeOut {
    print("time out")
}

异常转化为可选值

  • 结合可选类型，try？函数无异常返回值，反之为nil。
if let x = try? httpHandler(a: 409) {
    print("\(x)")
} else {
    print("error")
}

异常的隐式解析可选类型

  • 假装函数没有异常，但是真的遇到异常，就会报错了。
let x = try! httpHandler(a: 408)

指定清理

  • 使用defer在函数内部，最后执行，已完成结束操作，常用文件打开后的关闭函数。下面的例子比较随意，大概展示功能。
func deferHandler(a: Int) throws -> String {
    
    defer {
        print("end handler")
    }
    
    var sum = a
    for n in 1..<10 {
        sum = sum+n
        if sum > 999 {
            throw HttpResultError.Other
        }
    }
    
    return "ok"
}

let x = try? deferHandler(a: 998)
