模式

一个图案代表一个值或复合值的结构。例如，一元组的结构(1, 2)是逗号分隔的两个元素的列表。由于模式代表一个值，而不是任何一个特定值的结构，可以用各种价值观的匹配。例如，图案(x, y)的元组相匹配(1, 2)的任何其他两元素的元组。除了 ​​匹配用的值的图案，可以提取一个复合值的一部分或全部，每个部分到一恒定的或可变的名称绑定。

在斯威夫特，有两种基本模式：那些成功匹配任何类型的值，以及那些可能无法匹配在运行时指定的值。

第一种模式是用于简单的变量，常量和可选的绑定解构价值观。这些措施包括通配符模式，识别模式和有约束力的任何值或包含它们的元组模式。可以指定一种类型的注释为这些模式来约束它们符合特定类型的唯一值。

第二种模式是用于全模式匹配，其中的值你想匹配在运行时可能不存在。这些措施包括枚举例模式，可选的图案，表达模式和类型转换的模式。你在的情况下，标签使用这些模式switch的语句，一个catch一个的条款do声明，或在的情况下条件if，while，guard，或者for- in声明。

通配符模式

一个通配符模式匹配和忽略任何价值，由一个下划线（_）。当你不关心对价值被匹配使用通配符模式。例如，通过封闭范围以下代码迭代1...3，忽略对循环的每次迭代范围的电流值：

for _ in 1...3 {
// Do something three times.
}

标识图案

一个标识符模式匹配的任何值且匹配的值绑定到一个变量或常量名。例如，在下面的常量声明，someValue是该值相匹配的标识符图案42类型Int：

let someValue = 42
如果匹配成功，该值42是绑定的（分配），以恒定的名称someValue。

当在可变或恒定声明的左手侧上的图案是一个标识符图案，标识符图案是隐式的值结合图案的子模式。

值绑定模式

一个值绑定模式具约束力匹配的值到变量或常量名。值绑定结合匹配的值，以一个恒定的名字以begin模式let关键字; 那些结合到变量的名称开始与var关键字。

值绑定模式中标识的模式绑定新的命名变量或常量他们的匹配值。例如，可以分解一个元组的元素，每个元素的值绑定到对应的识别图案。

let point = (3, 2)
switch point {
// Bind x and y to the elements of point.
case let (x, y):
print(The point is at ((x), (y)).)
}
// Prints The point is at (3, 2).
在上面的例子中，let分配给在所述元组模式中的每个标识符图案(x, y)。由于这种行为的switch案件case let (x, y):和case (let x, let y):匹配相同的价值观。

元组模式

一个元组模式是一个逗号分隔的零个或多个模式列表，用括号括起来。元组的图案匹配相应的元组类型的值。

您可以约束一个元组模式通过使用类型注释某些类型的元组类型的匹配。例如，元组模式(x, y): (Int, Int)中的恒定声明let (x, y): (Int, Int) = (1, 2)匹配，其中两个元件是类型的唯一的元组类型Int。

当一个元组图谱以用作模式for- in语句或变量或常量声明，它只能包含通配符，标识图案，图案可选，或包含其他元组模式。例如，下面的代码是无效的，因为该元素0在元组模式(x, 0)是表达图案：

let points = [(0, 0), (1, 0), (1, 1), (2, 0), (2, 1)]
// This code isn't valid.
for (x, 0) in points {
/* ... */
}
围绕包含单个元件的元组图案括号没有任何效果。模式匹配单个元素的类型的值。例如，以下是等效的：

let a = 2 // a: Int = 2
let (a) = 2 // a: Int = 2
let (a): Int = 2 // a: Int = 2

可选模式

一个可选的模式匹配包裹在一个值some(Wrapped)的情况下Optional枚举。可选模式包括一个标识符模式之后紧接着一个问号出现在同一个地方作为枚举例模式。

因为可选的模式对于语法糖Optional枚举例模式，以下是等价的：

let someOptional: Int? = 42
// Match using an enumeration case pattern.
if case .some(let x) = someOptional {
print(x)
}

// Match using an optional pattern.
if case let x? = someOptional {
print(x)
}
可选模式提供了一个方便的方法来遍历可选值的一个阵列中的for- in声明，执行该循环体仅用于非nil元素。

let arrayOfOptionalInts: [Int?] = [nil, 2, 3, nil, 5]
// Match only non-nil values.
for case let number? in arrayOfOptionalInts {
print(Found a (number))
}
// Found a 2
// Found a 3
// Found a 5

表达模式

一个表达模式表示表达式的值。表达模式只出现在switch声明的情况下的标签。

通过表达模式表示的表达与使用夫特标准库的输入表达式的值进行比较~=运算符。比赛成功，如果~=运算符返回true。缺省情况下，~=操作者比较使用相同类型的两个值==运算符。它也可以用一个范围内的值相匹配的值，通过检查值是否包含在范围内，如以下示例所示。

let point = (1, 2)
switch point {
case (0, 0):
print((0, 0) is at the origin.)
case (-2...2, -2...2):
print(((point.0), (point.1)) is near the origin.)
default:
print(The point is at ((point.0), (point.1)).)
}
// Prints (1, 2) is near the origin.
您可以重载~=运营商提供定制的表达式匹配的行为。例如，你可以把上面的例子来比较point表达式点的字符串表示。

// Overload the ~= operator to match a string with an integer.
func ~=(pattern: String, value: Int) -> Bool {
return pattern == (value)
}
switch point {
case (0, 0):
print((0, 0) is at the origin.)
default:
print(The point is at ((point.0), (point.1)).)
}
// Prints The point is at (1, 2).
