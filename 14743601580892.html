词法结构

该词法结构斯威夫特的描述了字符序列形成的语言有效令牌。这些有效的令牌构成语言的最低层构建块，用于描述在后续章节语言的其余部分。令牌包含一个标识符，关键字，标点符号，文字或运营商。

在大多数情况下，是从夫特源文件的字符通过考虑从输入文本可能的最长子串，被下面指定的语法的约束内生成的标记。这种行为被称为最长匹配或最大适合。

空白和注释

空白有两种用途：源文件在不同的令牌，并帮助确定操作员是否是一个前缀或后缀（见运算符），否则忽略。以下字符被认为是空白：空格（U + 0020），换行符（U + 000A），回车（U + 000D），水平制表符（U + 0009），垂直制表符（U + 000B），换页（U + 000C）和空（U + 0000）。

评论被视为由编译器的空白。单行注释以开始//和继续下去，直到一个换行符（U + 000A）或回车（U + 000D）。多行注释开头/和结尾/。嵌套多行注释是允许的，但注释标记必须平衡。

注释可以包含其他格式和标记，如描述的标记格式参考。

身份标识

标识符以大写或小写字母A开头到Z，下划线（_），在基本多文种平面一个noncombining字母数字Unicode字符或基本多文种平面以外的字符不是在专用区。第一个字符后，数字和混合Unicode字符也是允许的。

要使用保留字作为标识符，把反引号（）之前和之后。例如，class是不是一个有效标识符，但class是有效的。反引号不考虑所述标识符的一部分;x`及x具有相同的含义。

与没有明确的参数名称的闭包内，参数隐式地命名\(0，\)1，$2，等。这些名称是封闭的范围内有效的标识符。

字面

一个文字是一个类型，值的源代码表示，如数字或字符串。

以下是面值的例子：

42 // Integer literal
3.14159 // Floating-point literal
Hello, world! // String literal
true // Boolean literal
字面上并没有对自己的一种类型。相反，字面解析为具有无限的精度和斯威夫特的类型推断会尝试推断适合文字类型。例如，在声明中let x: Int8 = 42，雨燕采用显式类型注解（: Int8）来推断整数文字的类型42是Int8。如果没有可用的合适类型的信息，斯威夫特推断字面的类型是在雨燕标准库中定义的默认文字类型之一。默认类型是Int整数常量，Double浮点文字，String为字符串文字和Bool布尔文字。例如，在声明let str = Hello, world中，默认推断类型的字符串的Hello, world是String。

当指定为文字值的类型注释，注释的类型必须是可以从字面数值实例化一个类型。也就是说，类型必须符合下列雨燕标准库协议之一：ExpressibleByIntegerLiteral字面上，整型变量，ExpressibleByFloatLiteral浮点文字，ExpressibleByStringLiteral对于字符串，ExpressibleByBooleanLiteral布尔文字，ExpressibleByUnicodeScalarLiteral对于只包含一个Unicode标字符串，并ExpressibleByExtendedGraphemeClusterLiteral为字符串仅包含单个扩展字形集群。例如，Int8符合ExpressibleByIntegerLiteral协议，因此它可以在类型标注为整数文字使用42在声明let x: Int8 = 42。

整数字面

整型常量代表未指定的精度整数值。默认情况下，整型常量在十进制表示; 你可以使用前缀指定备用基地。二进制文字开始0b，八进制文字首先0o，和十六进制文字开头0x。

十进制文本包含数字0通过9。二进制文字包含0和1，八进制文字包含0通过7和十六进制文字包含0通过9，以及A通过F在大写或小写。

负整数文字前面加上一个减号（表示-）字面一个整数，如-42。

下划线（_）的数字是为了便于阅读之间允许的，但它们将被忽略，因此不会影响到文字的值。整数文字可以带前导零的（开始0），但他们也同样被忽略，并且不影响文字的基础和价值。

除非另有规定，默认推断类型字面的整数的是夫特标准库类型Int。雨燕标准库还定义了类型符号和无符号整数的各种尺寸，如描述的整数。

浮点文本

浮点文本代表未指定精度浮点值。

缺省情况下，浮点文本在小数（没有前缀）来表示，但它们也可以以十六进制表示（具有0x前缀）。

十进制浮点文本由十进制数字后跟一个小数，小数指数，或两者序列。小数由小数点（中.），其次是十进制数字的序列。该指数由大写或小写字母的e前缀，后跟的十进制数字序列，表明权力的什么10前的值e乘以。例如，1.25e2代表1.25×10 2，计算结果为125.0。同样，1.25e-2代表1.25×10 -2，计算结果为0.0125。

十六进制浮点文本由的0x前缀，后面跟着一个可选的十六进制分数，后面跟十六进制指数。十六进制部分由一个小数点后面的十六进制数字序列。该指数由大写或小写字母的p前缀，后跟的十进制数字序列，表示的是什么力量2之前的值p乘以。例如，0xFp2代表15×2 2 ，计算结果为60。同样，0xFp-2代表15×2 -2，计算结果为3.75。

负浮点文字前面加上一个减号（表示-）到浮点文字，如-42.5。

下划线（_）的数字是为了便于阅读之间允许的，但将被忽略，因此不会影响到文字的值。浮点文本可以带前导零（开始0），但同样被忽略，并且不影响文字的基础和价值。

除非另有规定，默认推断类型浮点字面的是夫特标准库型Double，它表示64位浮点数。迅捷标准库还定义了Float类型，它表示32位浮点数。

运营商

雨燕标准库定义了一些运营商供您使用，其中许多在讨论基本运算和高级操作员。本节描述了字符可以用于定义自定义操作符。

运营商定制可以用ASCII字符中的一个开头/，=，-，+，!，*，%，<，>，&，|，，?，或者~，或者在下面的语法定义的Unicode字符（包括从人物的一个数学运算，杂项符号和装饰 Unicode的块，等等）。第一个字符后，结合Unicode字符也是允许的。

您还可以定义以点（开始运营商定制.）。这些运营商可以包含其他点如.+.。如果操作者不以点开始的，它不能包含一个点在其他地方。例如，+.+被视为+操作者随后.+操作者。

虽然你可以定义包含问号（运营商定制?），他们不能只包含一个问号字符。此外，尽管运营商可以包含一个感叹号（!），后缀运算符不能与任何一个问号或感叹号开始。

注意

该令牌=，->，//，/，/，.，前缀运营商<，&和?，管道符?和后缀运算符>，!和?保留。这些令牌不能被重载，也不能被用作运营商定制。

围绕操作员的空白被用来确定操作者是否被用作一个前缀操作员，一个后缀符，或二进制运算符。此行为总结于以下规则：

如果运营商已经围绕两侧或周围没有任何一方空白字符，它被视为一个二元运算。作为一个例子，+++在操作者a+++b和a +++ b被视为一个二进制运算符。
如果操作员对仅在左侧的空白，它被视为前缀元运算符。作为一个例子，在+++操作者在a +++b被视为前缀元运算符。
如果操作员对只有右侧的空白，它被当作一个后缀元运算符。作为一个例子，在+++操作者在a+++ b将被视为一个后缀元运算符。
如果操作员在左侧没有空白而是用点（紧跟.），它被视为一个后缀元运算符。作为一个例子，在+++操作者在a+++.b将被视为一个后缀元运算符（a+++ .b而非a +++ .b）。
对于这些规则的目的，人物(，[和{运营商之前，人物)，]和}运营商，和人物后,，;和:也被认为是空白。

有一点需要注意上述规则。如果!或?预定操作者在左侧没有空白，它被当作一个后缀符，而不管其是否具有在右侧的空白。要使用?作为可选链运营商，它不能在左边空白。在三元条件（使用）运营商，它必须有左右两侧的空白。 ? :

在某些构建体中，具有领先运营<或>可被分成两个或多个令牌。其余部分相同的方式处理，并且可以再次分裂。其结果是，没有必要使用空白闭合之间的歧义>在样构建的字符Dictionary>。在这个例子中，闭合>的字符不被视为其然后可以被误解为一个比特移位的单个令牌>>操作者。

要了解如何定义新的，运营商定制，看到运营商定制和运营商宣言。要了解如何重载现有的运营商，请参阅操作方法。
